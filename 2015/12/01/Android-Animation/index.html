Android Animation

发表于 2015-12-01   |   分类于 Android ， Android基础   |  

动画的类型

  • Property动画：可以对任意对象的属性（ObjectAnimator动画的属性要符合一定条件）进行动画。在Android 3.0（API 11）引入。
  • Tween动画：可以对View对象(或其他UI)的特定属性进行动画。
  • Drawable动画：可以一个接一个的显示Drawable对象，以形成动画。

Property动画

使用Property动画，需要提供setter方法（返回void），如果没有提供初始值或终止值，则必须提供getter方法。

Property动画执行流程

  1. 根据每一个Keyframe执行下列计算
  2. 根据Interpolator生成Fraction。
  3. TypeEvaluator根据Fraction生成值。
  4. 设置属性的值。

Property相关类

  • Animator：抽象类（也就是不能通过它对对象进行动画，而是通过它的子类），对动画提供了基本的支持。
  • ValueAnimator：Animator的子类。它只会生成Interpolated和相应的属性值，可以在Animator.AnimatorUpdateListener监听器中根据值改变对象的属性。
  • ObjectAnimator：ValueAnimator的子类。它要求提供要动画的对象和属性，然后调用start后自动更新对象的属性。
  • AnimatorSet：Animator的子类。它包含一组Animator，并按照一定次序播放动画。
  • LayoutTransition：动画效果是设置给容器（ViewGroup），然而效果是通过容器存放的View来体现的。
  • TimeAnimator：ValueAnimator的子类， There is no duration, interpolation, or object value-setting with this Animator。
  • TypeEvaluator及子类：提供用于改变对象属性的值。
  • TimeInterpolator及子类：生成Fraction。
  • PropertyValuesHolder：保存属性和属性在动画过程中的取值。
  • Keyframe：表示一对时间/值（某一时刻）。它有自己的Interpolator。
  • ViewPropertyAnimator：对View对象进行动画（更简单）。
  • StateListAnimator : 根据View的状态执行不同的动画
  • AnimatorInflater: 用来加载xml动画。

Property动画的使用

  • ValueAnimator

    1. 实例化（构造器或of…方法）并设置相应的属性，或使用AnimatorInflater.loadAnimator加载
    2. 添加AnimatorUpdateListener监听器
    3. 在监听器中调用getAnimatedValue得到值，并设置为要动画的对象的属性值。
    4. 在需要时调用start方法启动。

  • ObjectAnimator

    1. 实例化（构造器或of…方法）并设置相应的属性，它要求提供要动画的对象和属性
    2. 或者使用AnimatorInflater.loadAnimator加载xml动画，并调用setTarget方法设置对象
    3. 在需要时调用start启动。

  • AnimatorSet

    1. 实例化AnimatorSet，或使用AnimatorInflater.loadAnimator加载
    2. 以特定顺序组合AnimatorSet
    3. 在需要时调用start启动。

  • LayoutTransition

    1. 实例化LayoutTransition并设置相应的属性
    2. 给每种变化设置Animator（可以调用LayoutTransition.getAnimator得到默认的Animator）
    3. 调用ViewGroup对象的setLayoutTransition设置LayoutTransition。
    4. 当相应的事件发生时，动画自动运行。
    5. 或者设置ViewGroup的xml属性 animateLayoutChanges="true"以使用默认动画。

  • ViewPropertyAnimator

    1. 调用View对象的animate返回ViewPropertyAnimator。
    2. 设置要动画的属性
    3. 在View显示时，自动开始动画。或者调用start立即开始动画

  • StateListAnimator

    1. 实例化，并增加相应状态；或者调用AnimatorInflater.loadStateListAnimator加载
    2. 调用View对象的setStateListAnimator设置动画

Tween动画

Tween动画原理

  1. 当View invalidate时，会向上传递给它的parent。
  2. 当parent调用重绘时，如果发现子View有动画，则调用Animation的initialize方法让Animation根据参数初始化。
  3. 然后调整子View Canvas的坐标系或透明度（根据Animation的getTransformation方法调整），然后重绘子View，这样就画出了动画的一帧
  4. 如果getTransformation返回true，则调用invalidate继续重绘，而继续重绘导致会继续调整Canvas的坐标系或透明度（因为getTransformation随着时间返回的数据不同），这样就绘制出了另一帧
  5. 调到第3步，直到动画完成（getTransformation返回false）。

从上面可知，Tween动画并没有改变View内部的属性。

Tween动画相关类

  • Animation：抽象类。对Tween动画提供了基本支持。
  • Animation子类：包括平移、旋转、缩放、透明度和AnimationSet。
  • LayoutAnimationController及子类：对ViewGroup中的children进行动画。每个child的动画相同，但开始时间不同。
  • AnimationUtils：提供实用方法、加载动画/Interpolater和一些预置动画。
  • TimeInterpolator及子类：生成Fraction。

Tween动画的使用

  • Animation子类

    1. 实例化，并设置相应的属性；或者使用AnimationUtils中的方法加载
    2. 设置开始时间：Animation.setStartTime，应该通过AnimationUtils.currentAnimationTimeMills来表示当前毫秒，或者使用Animation.START_ON_FIRST_FRAME。
    3. 调用View对象的setAnimation设置。
    4. 或者把第2,3步替换为：调用View对象的startAnimation。

  • LayoutAnimationController及子类

    1. 实例化，并设置相应属性；或者使用AnimationUtils中的方法加载
    2. 调用ViewGroup对象的setLayoutAnimation设置。
    3. 或者直接在layout文件中指定： android:layoutAnimation="@anim/list_item_layout"
