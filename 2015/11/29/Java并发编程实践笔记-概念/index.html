Java并发编程实践笔记-概念

并发与并行

  • 并发（concurrent）：多个线程在同一个CPU上交替执行。
  • 并行（parallel）：多个线程在各自的CPU上执行。

多线程的优势

  • 发挥多处理器的强大能力（即使在单处理器上也能获得更高的吞吐率）。
  • 建模的简单性（为每个任务分配一个线程，并在需要时进行交互）。
  • 异步事件的简单处理（如服务器对每个请求创建一个线程（一般有最大线程数量的限制））。
  • 响应更灵敏的用户界面

多线程带来的风险

  • 安全性：产生正确的结果
  • 活跃性：正确的结果最终会发生（如死锁）
  • 性能：正确的结果尽快发生

最低安全性

当线程在没有同步的情况下读取变量时，可能是一个失效值，但至少这个值是由之前某个线程设置的值，而不是一个随机值。

最低安全性适用于大多数变量，但存在一个例外：非volatile类型的64位数值变量（double和long）。使用volatile可以保证它们的最低安全性。
